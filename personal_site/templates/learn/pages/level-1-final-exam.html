Answer Key

  1. a
  2. Typical answers: int, float, str, bool, list, tuple, dict
  3. c
  4. c (Explanation: In python3, division defaults to floating point division)
  5. a (Explanation: // means integer division and truncates the decimal)
  6. True
  7. True
  8. c
  9. Python uses indentation to mark the start and end of a function
  10. Functions are defined by using def func_name(parameter_list):. When calling a function, you omit the def and :. Furthermore, a function must have been previously defined before it can be called.
  11. See below* (I don't want to screw up inline formatting)
  12. c (Explanation: As soon as the first True conditional is reached, execution continues after the entire conditional block)
  13. input() returns a str by default, so without an explicit cast with int() or float(), it will just print the string twice (so 2 -> 22)
  14. a
  15. a
  16. d (Explanation: the parameter to range is the upper limit, but the range will not go up to the value; it stops before it. So our range is [0, 1, 2, 3, 4, 5, 6, 7]. When filtering to numbers that num % 2 == 0 (even numbers), we are just adding [0, 2, 4, 6])
  17. d (Explanation: We never advance index so the program will continue forever -- we call this an infinite loop)
  18. When you don't know how many times you will loop -- example, getting input from the user until they give you a valid input
  19. When you know exactly how many times you will loop -- example, iterating over the values in a list
  20. b
  21. b (Explanation: keys are unique, so we overwrote the initial value of "Smith")
  22. A tuple is immutable -- it can't be resized or modified ever
  23. d
  24. read, write, append
  25. The file contents are completely erased first, then new contents are written.
