Exercises

  • [circle] Write a Circle class that has a center point and a radius.
  • [vector] Write a Vector class that represents a 2D vector (If you'd like a quick refresher on vectors, check out this Wikipedia article, but you don't have to be an expert). A vector has both direction and magnitude. Assume that our vectors always originate at the origin (point (0, 0)) and have a single terminal point. Create functions vector_length(v), vector_add(v1, v2), vector_sub(v1, v2), vector_scale(v, d) (scalar multiplication), and vector_dot(v1, v2). You can find definitions for these operations here.

NOTE: You may also want to define a Point class for this exercise.

Project Part 1

For this project, we will be creating a terminal-based Sudoku game. As part 1 of this project, you should create a new class SudokuGame that stores the current state of the game Recall that a sudoku game is played on a 9x9 grid. For this first part, you can assume the board is initialized as completely blank (pretty useless for a real sudoku game, but this will be okay for now).

We'll also create the game setup here. A user should run your program with python sudoku.py. Upon startup, ask the user what difficulty they'd like to play: easy, medium, or hard. You should have three folders in your game directory: easy_boards, medium_boards, and hard_boards. After the user makes a selection, pick a random game board for them to play. We don't necessarily need to think about how to load this as a new SudokuGame object yet. Just select the correct file for now.

In this first step, our game won't be very fun, but that's okay. We'll build it out in parts 2 and 3 to provide more functionality.

INSTRUCTIONS BEFORE STARTING:

You are encouraged to start from a completely blank file so you can get more exposure to starting projects from scratch, but if you're not sure where to begin, feel free to start from the template I've provided on my bitbucket page here. This is the skeleton code for part 1.

NOTE: If you'd like my solution for this part of the project, you can check out my solution here. Please try not to look until you've attempted this yourself, though!

You should also download this file and run unzip sudoku_boards.zip in the terminal to get a bunch of premade sudoku boards.

Here are the objectives for part 1:

  • Create SudokuGame class with relevant fields
  • Ask the user what difficulty they'd like to play
  • Pick a random game board from that difficulty
  • Parse the gameboard file into a list of lists (this is how we can represent a 2D game board). See the example below for how this would work.
# Imagine the game board text file looks like this:
# (Don't worry about this being an invalid board for now)

7 8 9 1 2 3 4 5 6

"""

Bonuses:
