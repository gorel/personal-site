Quiz

Use the following program to answer questions 1-3.

class Point:
    """Represents a point in 2D space"""
    def __init__(self, x, y):
      self.x = x
      self.y = y

    def __str__(self):
      return f"Point(x = {self.x}, y = {self.y})"

  p1 = Point(3, 4)
  p2 = Point(y=2, x=1)

  1. What would happen if you typed print(p1)?
    a. Point(x = 3, y = 4)
    b. Point(3, 4)
    c. <Point at 0x7eff85415980> (or something similar)
    d. (error)

  2. What do we call methods such as __init__ or __str__?
    a. underscore functions
    b. magic methods
    c. funky functions
    d. quick methods

  3. How can you implement equality checking (==) for a user-defined class?
    a. def __equality__(self, other):
    b. def equals(a, b):
    c. def ==(self, other):
    d. def __eq__(self, other):

Project Part 2

In part 2, we are going to create the "main game loop." In the game loop, this is what should happen:
1. print out the current state of the board
2. ask the user for an input. Input can either be "solve1" (computer will randomly fill in 1 correct answer), "solveall" (computer will complete the entire board), or input like "row col val" where the user wishes to set coordinate (row, col) to the value val. Example: 1 1 4 would set the space (1, 1) to 4.
3. update the game board given the last input
4. If the board is completely solved, end the game and tell the user how long they took in seconds to complete the game.

INSTRUCTIONS BEFORE STARTING:

You are encouraged to continue this project from what you made in part 1, but if you're having trouble extending your project, feel free to continue from the template I've provided on my bitbucket page here. This is the skeleton code for part 2.

If you didn't do so in part 1, you should also download this file and run unzip sudoku_boards.zip in the terminal to get a bunch of premade sudoku boards.

Here are the objectives for part 2:
- Add methods to SudokuGame class for __init__ (if you didn't do so in part 1), __str__(), is_solved(), solve_one(), solve_all(), and set(row, column, value)
- Create a main game loop that repeatedly asks the user for input

Bonuses:
1. Add a "check" input that will remove any wrong guesses from the current board state
2. Add an index to the x and y axis so the user can easily see what row and column they wish to fill in next
3. Add color! Have starting values in the grid be white, user input be blue, and if you've added the "check" command, have correct cells be green and incorrect cells be red after a call to check.
